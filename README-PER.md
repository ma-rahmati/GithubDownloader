# 📥 دانلودر گیت‌هاب

این یک ابزار مبتنی بر GitHub Actions است که فایل‌ها را از طریق لینک داده شده دانلود کرده و آن‌ها را به صورت خودکار در **GitHub Releases** آپلود می‌کند، همراه با پشتیبانی از تقسیم فایل‌های بزرگ (ZIP Split).

---

## 🚀 قابلیت‌ها

* 📥 دانلود فایل از چندین لینک همزمان
* ⚡ دانلود سریع با aria2 (چند کانکشن)
* 📦 فشرده‌سازی خودکار فایل‌ها (ZIP)
* ✂️ تقسیم فایل‌های بزرگ برای محدودیت GitHub
* 🚀 آپلود مستقیم در GitHub Releases
* 🏷️ امکان تعیین نام دلخواه برای Release
* 🧠 تولید خودکار نام در صورت عدم وارد کردن

---

## ⚙️ نحوه عملکرد

1. کاربر یک یا چند لینک وارد می‌کند
2. فایل‌ها با aria2 دانلود می‌شوند
3. هر فایل به ZIP تبدیل می‌شود
4. فایل‌های بزرگ به بخش‌های کوچک تقسیم می‌شوند
5. همه فایل‌ها در GitHub Releases آپلود می‌شوند

---
⚙️ نحوه استفاده (مهم)

⚠️ قبل از شروع باید از این پروژه Fork بگیرید (یک کپی از ریپازیتوری در حساب GitHub خودتان).

### 1. فورک کردن پروژه

وارد صفحه اصلی ریپازیتوری شوید

روی دکمه Fork کلیک کنید (بالای صفحه سمت راست)

یک کپی در حساب GitHub شما ساخته می‌شود

---
### 2. ورود به ریپازیتوری Fork شده

وارد نسخه‌ای شوید که در حساب خودتان ساخته شده است.

---

### 3. رفتن به بخش Actions

```
Actions → Download via GitHub
```

---

### 4. اجرای Workflow

روی گزینه زیر کلیک کنید:

```
Run workflow
```

---

### 📌 ورودی‌ها

| فیلد         | توضیح                               |
| ------------ | ----------------------------------- |
| urls         | لیست لینک‌ها با فاصله               |
| split_mb     | حداکثر حجم هر بخش (پیش‌فرض: 1900 مگابایت) |
| release_name |  نام دلخواه برای ریلیز کردن فایل (اختیاری)   |

---

### 🧾 مثال ورودی

```
https://example.com/file1.iso https://example.com/file2.zip
```

---

## 📦 خروجی

پس از اجرا، فایل‌ها در این بخش قرار می‌گیرند:

```
GitHub → Releases → Latest Release
```

شما ممکن است فایل‌های زیر را ببینید:

* فایل‌های .zip
* فایل‌های .z01, .z02 (در صورت تقسیم)

---

## 🧩 نحوه استخراج فایل‌ها

### 💻 در ویندوز (پیشنهادی)

نصب کنید:

* 7-Zip

سپس:

1. تمام فایل‌ها را دانلود کنید
2. روی فایل `.zip` راست کلیک کنید
3. گزینه Extract Here را بزنید
4. تمام

---

### 🐧 در لینوکس / مک

اگر فایل split شده باشد:

```bash
zip -s 0 file.zip --out full.zip
unzip full.zip
```

---

## 📝 نکات:

* محدودیت GitHub برای هر فایل حدود 2GB است
* فایل‌های بزرگ به صورت خودکار تقسیم می‌شوند
* فایل‌های `.z01` و `.z02` را حذف نکنید
* نام Release اختیاری است
